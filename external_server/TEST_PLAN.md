# Sniffer Client SDK Test Plan

This document outlines the test strategy for the `sniffer_client` SDK library (`libsniffer_client.so`).

## 1. Test Environment
*   **Build Directory:** `build_external_service` (separate from main build).
*   **Components:**
    *   `car_system_emulator` (vcan0)
    *   `car_computer_emulator` (vcan1)
    *   `RpiCanbusSniffer` (Bridge vcan0 <-> vcan1)
    *   `sniffer_client_test.py` (Test Runner using the SDK)

## 2. Test Cases

### 2.1 Lifecycle Management
*   **Initialization:** Verify `client_create` returns a valid handle.
*   **Startup:** Verify `client_start` returns success (0).
*   **Shutdown:** Verify `client_stop` and `client_destroy` clean up resources without crashing.
*   **Restart:** Verify the client can be stopped and started again.

### 2.2 Logging & Data Reception
*   **Enable Logging:** Send `client_send_log_enable(true)`.
*   **Receive Data:** Call `client_read_message` and verify reception of CAN frames generated by emulators.
*   **Verify Content:** Check that received frames have correct Magic Key ("canS" / "canC") and valid CAN IDs (e.g., 0x100, 0x200).
*   **Disable Logging:** Send `client_send_log_enable(false)` and verify no new messages are received.

### 2.3 Injection
*   **Inject to System:** Send a frame to `TO_SYSTEM` (e.g., ID 0x400). Verify it appears on `vcan0`.
*   **Inject to Car:** Send a frame to `TO_CAR` (e.g., ID 0x7DF). Verify it appears on `vcan1`.

### 2.4 Filter Management
*   **Set Drop Rule:** Create a rule to DROP ID 0x200 (Speed). Apply via `client_set_filters`.
*   **Verify Drop:** Confirm 0x200 frames are no longer received on the destination bus.
*   **Set Modify Rule:** Create a rule to MODIFY ID 0x100 (RPM). Apply.
*   **Verify Modify:** Confirm 0x100 frames arrive with modified data.
*   **Clear Filters:** Send empty filter list. Verify normal traffic resumes.

### 2.5 Keep-Alive Mechanism
*   **Automatic Sending:** Configure client with short keep-alive interval (e.g., 100ms).
*   **Verification:** Monitor UDP traffic (using a separate socket in the test script) to ensure `CMD_CANBUS_DATA` packets are sent periodically when idle.

### 2.6 Error Handling & Robustness
*   **Invalid Handle:** Call API functions with NULL handle. Verify no crash.
*   **Sniffer Offline:** Start client while Sniffer is down. Verify graceful error handling.
*   **Reconnection:** Simulate Sniffer restart. Verify client can resume communication.

## 3. Execution
The tests will be executed via `run_all_tests.py` (or a dedicated script) which will:
1.  Build the SDK.
2.  Setup vcan interfaces.
3.  Launch emulators and Sniffer.
4.  Run the Python test script.
5.  Report Pass/Fail status.
